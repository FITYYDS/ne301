# NE301 产品能力总图

## 1. 一句话理解

NE301 不是“一个能拍照的 AI 摄像头”这么简单，它本质上是一套把感知、联网、视频、AI、运维和升级能力，统一收敛到同一套 MCU 服务框架里的产品平台。

## 3. 这张图怎么读

从上往下看：

1. 最上面是用户真正能感知到的产品能力。
2. 中间是这些能力共用的统一软件骨架，也就是 `api_xxx -> service_xxx -> config/persist -> HAL`。
3. 最下面是支撑这些能力的底座，包括视频链路、网络链路、存储链路，以及 `U0 + N6` 的双 MCU 架构。

从左往右看：

1. 左边偏“触发与控制”。
2. 中间偏“联网与云协同”。
3. 右边偏“视频与 AI 数据面”。
4. 下方单独汇总“设备运维与升级”。

## 4. 这张总图对应的核心判断

- NE301 的很多功能虽然在 wiki 上看起来是分散菜单，但在软件上并不是很多孤立模块，而是同一套服务框架的不同投影。
- `system_service` 是控制面中枢，`communication_service` 是网络中枢，`video_hub` 是视频数据分发中枢。
- 产品功能的扩展方式不是继续堆页面，而是往现有服务框架里接入新的策略、事件源和消费者。
